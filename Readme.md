# PowerShell WebP Converter

🖼️ **Скрипт для пакетной рекурсивной конвертации изображений в формат WebP**

Этот PowerShell-скрипт автоматически находит все изображения в указанной папке (и всех вложенных папках) и конвертирует их в современный формат **WebP** с помощью утилиты `cwebp` (Google WebP).

## ✨ Возможности

- 🔄 **Рекурсивная обработка:** Сканирует текущую папку и все подпапки.
- 🎛️ **Настройка качества:** Возможность указать уровень сжатия (0–100).
- 🗑️ **Опциональное удаление:** Возможность удалить оригиналы после успешной конвертации.
- 📊 **Прогресс-бар:** Отображение статуса обработки в реальном времени.
- 🛡️ **Безопасность:** По умолчанию оригиналы не удаляются; игнорирует уже существующие `.webp` файлы.
- 📁 **Поддержка форматов:** `.png`, `.jpg`, `.jpeg`, `.bmp`, `.tiff`, `.tif`.

## 📋 Требования

1. **PowerShell** версии 5.1 или выше (рекомендуется PowerShell 7+).
2. **Утилита `cwebp`**:
   - Скрипт не содержит саму утилиту конвертации.
   - Необходимо установить пакет **libwebp**.
   - **Windows:** [Скачать здесь](https://developers.google.com/speed/webp/download), распаковать и добавить папку `bin` в системную переменную `PATH`.
   - **Linux:** `sudo apt install webp` (Debian/Ubuntu) или `sudo dnf install libwebp-tools` (Fedora).
   - **macOS:** `brew install webp`.

## 🚀 Быстрый старт

### 1. Установка

Склонируйте репозиторий или скачайте файл `Convert-ToWebP.ps1`:

```bash
git https://github.com/webton-labs/convert-towebp.git
cd convert-towebp
```

### 2. Подготовка PowerShell

По умолчанию Windows может блокировать выполнение скриптов. Разрешите выполнение для текущей сессии:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

### 3. Запуск

Перейдите в папку с изображениями, положите в нее файл `Convert-ToWebP.ps1` и запустите скрипт:

```powershell
.\Convert-ToWebP.ps1
```

## ⚙️ Параметры скрипта

| Параметр | Тип | По умолчанию | Описание |
| :--- | :--- | :--- | :--- |
| `-Path` | `String` | `.` (текущая) | Путь к папке с изображениями для обработки. |
| `-Quality` | `Int` | `90` | Качество выходного файла (от 0 до 100). |
| `-DeleteOriginal` | `Switch` | `False` | **Внимание!** Удалять оригинальные файлы после конвертации. |
| `-CwebpPath` | `String` | `cwebp` | Полный путь к `cwebp.exe`, если утилита не добавлена в `PATH`. |

## 💡 Примеры использования

**Конвертация изображений в текущей папке (качество 80):**
```powershell
.\Convert-ToWebP.ps1
```

**Конвертация конкретной папки с качеством 80:**
```powershell
.\Convert-ToWebP.ps1 -Path "C:\Users\Name\Pictures" -Quality 80
```

**Конвертация с удалением оригиналов (использовать с осторожностью!):**
```powershell
.\Convert-ToWebP.ps1 -Path "D:\Images" -DeleteOriginal
```

**Если утилита cwebp не в PATH:**
```powershell
.\Convert-ToWebP.ps1 -CwebpPath "C:\Tools\libwebp\bin\cwebp.exe"
```

## ⚠️ Важные предупреждения

1. **Резервное копирование:** Перед использованием флага `-DeleteOriginal` настоятельно рекомендуется сделать резервную копию данных.
2. **Зависимость:** Убедитесь, что команда `cwebp` доступна в терминале. Проверить можно, введя `cwebp -version` в PowerShell.
3. **Перезапись:** Скрипт создает файлы с расширением `.webp` рядом с оригиналами. Если файл `.webp` уже существует, он будет перезаписан.

## 🛠️ Устранение неполадок

**Ошибка: "Утилита 'cwebp' не найдена"**
- Убедитесь, что libwebp установлен.
- Добавьте путь к папке `bin` libwebp в переменную среды `PATH`.
- Либо используйте параметр `-CwebpPath` для указания полного пути к exe-файлу.

**Ошибка: "Выполнение скриптов отключено"**
- Запустите PowerShell от имени администратора и выполните:
  `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
- Или используйте обход для текущей сессии (см. раздел "Быстрый старт").
